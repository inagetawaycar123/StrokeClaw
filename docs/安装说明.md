# StrokeClaw 安装说明

本文档用于从零部署并运行 StrokeClaw 作品。StrokeClaw 由 Flask 后端、React/Vite 前端、医学影像推理模型、知识库与报告生成模块组成。

## 1. 运行环境

### 1.1 基础软件

请先安装以下软件：

| 软件 | 建议版本 | 说明 |
|---|---:|---|
| Python | 3.14 或与 `.python-version` 一致 | 后端服务、推理脚本、测试运行环境 |
| uv | 最新稳定版 | Python 依赖与虚拟环境管理 |
| Node.js | 18+ | 前端构建环境 |
| npm | 随 Node.js 安装 | 前端依赖安装与打包 |
| Git | 最新稳定版 | 获取代码与版本管理 |
| CUDA / NVIDIA 驱动 | 按本机 GPU 匹配 | 使用 GPU 推理时需要 |

如果没有 GPU，也可以使用 CPU 运行部分功能，但医学影像模型推理速度会明显变慢，部分大模型功能可能无法正常加载。

### 1.2 推荐硬件

- Windows 10/11 或 Linux。
- 至少 16 GB 内存；如果加载 MedGemma 等大模型，建议 32 GB 以上。
- NVIDIA GPU 建议 8 GB 显存以上；模型越完整，显存需求越高。
- 预留足够磁盘空间存放模型权重、上传病例和运行缓存。

## 2. 获取代码

进入准备存放项目的目录，克隆或解压作品代码：

```bash
git clone <your-repository-url> StrokeClaw
cd StrokeClaw
```

如果代码已经在本机，例如当前目录为 `G:\StrokeClaw`，可直接进入项目根目录：

```powershell
cd G:\StrokeClaw
```

## 3. 安装 Python 依赖

项目根目录包含 `pyproject.toml` 与 `uv.lock`，推荐使用 `uv` 安装后端依赖：

```bash
uv sync
```

如果需要单独使用 `pip` 安装，可使用：

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Linux/macOS 下激活虚拟环境：

```bash
source .venv/bin/activate
```

注意：`pyproject.toml` 中配置了 PyTorch CUDA 依赖源。如果本机 CUDA 版本与项目配置不一致，请根据实际显卡驱动重新安装匹配的 `torch` 与 `torchvision`。

## 4. 安装前端依赖并构建

进入前端目录：

```bash
cd frontend
npm install
npm run build
cd ..
```

构建完成后，前端产物会生成到 Vite 配置指定的输出目录，并由 Flask 后端提供页面访问。

开发调试前端时可单独运行：

```bash
cd frontend
npm run dev
```

默认前端开发服务会代理到后端：

```env
VITE_PROXY_TARGET=http://127.0.0.1:5011
```

## 5. 配置环境变量

在项目根目录创建 `.env` 文件。可参考下列内容：

```env
FLASK_ENV=development
FLASK_DEBUG=1

VITE_API_URL=http://127.0.0.1:5011
VITE_API_BASE_URL=http://127.0.0.1:5011

BAICHUAN_API_URL=https://api.baichuan-ai.com/v1/chat/completions
BAICHUAN_API_KEY=your_api_key_here
BAICHUAN_MODEL=Baichuan-M3
BAICHUAN_CHAT_MODEL=Baichuan-M3

KB_PDF_DIR=static/kb
EKV_DOCS_DIR=EKV_docs
STROKE_KG_PATH=runtime/kg/stroke_kg.json

MODEL_WARMUP_ASYNC=1
MODEL_WARMUP_WAIT_TIMEOUT_MS=12000
MODEL_WARMUP_CTP_TIMEOUT_MS=120000
```

其中：

- `BAICHUAN_API_KEY` 用于报告生成、问答和知识库相关能力；未配置时部分功能会降级或返回模拟内容。
- `KB_PDF_DIR` 指向内置知识库 PDF 目录。
- `EKV_DOCS_DIR` 指向指南与循证知识文档目录。
- `STROKE_KG_PATH` 指向卒中知识图谱 JSON。
- `MODEL_WARMUP_ASYNC` 控制模型是否异步预热。

请不要把真实 API Key 提交到公开仓库。

## 6. 放置模型权重

部分大文件权重通常不会直接放入 Git 仓库，需要手动下载后放到指定位置。

| 模块 | 需要的文件或目录 | 放置位置 |
|---|---|---|
| MedGemma | `model-00001-of-00002.safetensors`、`model-00002-of-00002.safetensors` 等 | `MedGemma_Model/` |
| NCCT 三分类模型 | `best_model.pt` | `backend/three_class/best_model.pt` |
| DINOv3 血管/灌注相关模型 | `dinov3权重.pth` | `dinov3/src/dinov3权重.pth` |
| DINOv3 预训练模型 | `dinov3_vitb16_pretrain_lvd1689m-73cec8be.pth` | `dinov3/src/ckpt/` |
| Palette CTP 模型 | CBF、CBV、Tmax 权重目录 | `palette/weights/cbf/`、`palette/weights/cbv/`、`palette/weights/tmax/` |
| MRDPM 模型 | CBF、CBV、Tmax 权重目录或对应 `.pth` 文件 | `mrdpm/weights/` |

放置完成后，建议检查目录：

```bash
ls MedGemma_Model
ls backend/three_class
ls dinov3/src
ls palette/weights
ls mrdpm/weights
```

Windows PowerShell 可使用：

```powershell
Get-ChildItem MedGemma_Model
Get-ChildItem backend\three_class
Get-ChildItem dinov3\src
Get-ChildItem palette\weights
Get-ChildItem mrdpm\weights
```

## 7. 启动后端服务

在项目根目录运行：

```bash
uv run python run.py
```

如果使用手动创建的虚拟环境：

```bash
python run.py
```

服务默认监听：

```text
http://127.0.0.1:5011
```

浏览器访问该地址即可进入系统。

## 8. 验证安装

### 8.1 后端基础检查

启动后端后，确认终端没有关键报错，并访问：

```text
http://127.0.0.1:5011
```

如页面能够正常打开，说明 Flask 服务和静态页面加载成功。

### 8.2 前端构建检查

重新执行：

```bash
cd frontend
npm run build
cd ..
```

构建无报错即可。

### 8.3 单元测试

项目包含若干后端测试，可在根目录运行：

```bash
uv run pytest
```

或运行核心测试：

```bash
uv run pytest tests/test_agent_loop_modules.py tests/test_icv.py tests/test_ekv.py
```

## 9. 常见问题

### 9.1 `Model checkpoint not found`

说明模型权重没有放到代码期望的位置。请按照“放置模型权重”章节检查文件路径，例如：

```text
backend/three_class/best_model.pt
dinov3/src/dinov3权重.pth
dinov3/src/ckpt/dinov3_vitb16_pretrain_lvd1689m-73cec8be.pth
```

### 9.2 `BAICHUAN_API_KEY` 未配置

报告生成、实时问答或知识库相关功能可能不可用。请在 `.env` 中设置：

```env
BAICHUAN_API_KEY=your_api_key_here
```

### 9.3 PyTorch 或 CUDA 不匹配

如果出现 CUDA、GPU、`torch` 加载错误，请先确认：

```bash
python -c "import torch; print(torch.__version__); print(torch.cuda.is_available())"
```

若返回 `False`，请根据本机 CUDA 版本重新安装对应 PyTorch。

### 9.4 端口 `5011` 被占用

关闭占用端口的程序，或在 `run.py` 中调整 Flask 启动端口后重新运行。

### 9.5 Windows PowerShell 显示中文乱码

可先切换终端编码：

```powershell
chcp 65001
```

并确保编辑器以 UTF-8 打开 Markdown 文档。
